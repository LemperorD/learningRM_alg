**TRANSISTOR**

---
这份作业旨在通过一个简单的 C++ 项目，带你走通一个简单完整工作流程：
代码编写 -> 构建系统 -> 脚本自动化 -> 版本控制 -> 文档规范。

在 Transistor，这样的任务到处都是，因此挑战一下自己，尽量在不使用 ai **“一步到位"** 的情况下，完成本次作业。


## 1. 目标
1.  Git：掌握 git 的简单操作，理解 github 的使用，远程与本地的关系。
2.  CMake：学习管理多文件夹结构的工程构建。
3.  Bash：编写简单的脚本，实现一键编译运行。
4.  Markdown：试着编写标准、专业的项目说明文档 (README.md)。让别人看懂你的代码，是作为程序员最基本也是最重要的技能。

---

## 2. 第一阶段：Git
请**不要**直接在我的仓库里提交代码，你需要建立一个属于自己的仓库。

1.  克隆模板
2.  切断关联：删除指向远程仓库的连接。
    ```bash
    git remote remove origin
    ```
3.  建立个人仓库：
    *   在 GitHub 上新建一个名为 `cpp-factorial-tool` 的空仓库。
    *   将你的本地文件夹关联到这个新仓库：
    ```bash
    git remote add origin https://github.com/你的用户名/cpp-factorial-tool.git
    ```

## 3. 第二阶段：工程目录组织
你需要手动创建以下目录结构（试着用用终端指令），严禁将所有文件堆放在根目录：

```text
.
├── include/         # 存放头文件 (.hpp)
├── src/             # 存放源代码 (.cpp)
├── CMakeLists.txt   # CMake 配置文件
├── run.sh           # 自动化 Bash 脚本
├── .gitignore       # Git 忽略规则
└── README.md        # 项目文档
```
BTW：终端中输入下列指令可以让你看到类似上图的图像

```bash
sudo apt install tree
tree .
```

### 代码要求：
*   `include/factorial.h`：声明阶乘计算函数 `long long calculate(int n);`。
*   `src/factorial.cpp`：实现上述函数逻辑，并 include 上述文件。
*   `src/main.cpp`：调用函数并在终端打印输出用户输入的数的阶乘。
*  本作业并不着重考察算法能力，所以能用就行。

## 4. 第三阶段：构建与自动化配置

### 1. CMakeLists.txt
要求使用 CMake 管理工程，核心配置必须包含：
```cmake
cmake_minimum_required(VERSION 3.10)
project()#你起的名字
#你应该完成的部分
```
 不要手写 build 文件夹：让 CMake 自动生成它，你的代码库应该保持干净。

### 2. run.sh（自动化脚本）
编写一个脚本，要求输入 `bash run.sh` 后自动完成以下动作：
1.  调用 `cmake -B build` 生成构建环境。
2.  调用 `cmake --build build` 进行编译。
3.  运行生成的可执行文件 `./build/你起的名字`。

### 3. .gitignore（环境清理）
确保 `build/` 文件夹内的中间文件不会被提交，编写 .gitignore 来实现这一点。

## 5. 第四阶段：编写 README.md

一份及格的文档应包含：
1.  项目简介：简单描述这是什么。
2.  项目树展示：使用 Markdown 代码块展示你的目录结构。
3.  如何运行：写明环境要求及运行命令（`bash run.sh`）。
4.  \*可以在readme最后提问，也可以直接在群里问或者私信。欢迎问题，但在那之前，可以耐心看看 [提问的智慧]([How-To-Ask-Questions-The-Smart-Way/README-zh_CN.md at main · ryanhanwu/How-To-Ask-Questions-The-Smart-Way](https://github.com/ryanhanwu/How-To-Ask-Questions-The-Smart-Way/blob/main/README-zh_CN.md))。

3.  将你的 GitHub 仓库链接 发送到指定位置。

## 提示
*   遇到报错不要慌：先看看哪里在报错。这是个小项目，相信你能根据报错找到问题。
*   关于脚本权限：如果 `run.sh` 无法运行，尝试输入 `chmod +x run.sh` 赋予它执行权限。
